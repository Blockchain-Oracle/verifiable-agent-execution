## Track 1 competitors — detailed profiles

### 0G OpenClaw
- **Track:** 1 (Agentic Infrastructure & OpenClaw Lab)
- **What it does:** A personal OpenClaw instance wallet-bound to your $0G wallet. Chat history encrypted and stored on 0G Storage. Models powered by 0G Compute Network. Cross-device memory retrieval by importing wallet. Built on top of OpenClaw, retains full OpenClaw feature set.
- **Tech stack:** Web3, Agent, Node, Rust, TypeScript
- **Deploy ecosystem:** 0G
- **Demo video:** Exists (mp4 link on project page)
- **Maturity signal:** Has demo video + project images. "Built on top of OpenClaw" — not a fork, a wallet-gated wrapper. Describes full implementation of the Storage + Compute integration.
- **Lane overlap with Target Track 1:** HIGH — this is the most direct competitor. It does: OpenClaw + 0G Storage memory + 0G Compute inference. This covers Wedge #1 and Wedge #2 from prior research.
- **Key gap in this project:** No on-chain audit trail / verifiable multi-agent handoff. No iNFT integration. No 0G Private Computer (launched today, post-submission).
- **Builder:** Lewis Gao (avatar-default-01, no additional public profile found)

### AgentMem
- **Track:** 1 (Agent memory infrastructure)
- **What it does:** Decentralized Knowledge Infrastructure for Autonomous Agents. Full description is minimal ("Decentralized Knowledge Infrastructure for Autonomous Agents") — very early or underdeveloped submission.
- **Tech stack:** Next, React, Web3, Solidity
- **Deploy ecosystem:** Arbitrum One — NOT 0G mainnet. This is a potential disqualification risk under the 0G integration requirement.
- **Maturity signal:** Thin description, no progress notes, deploys to Arbitrum not 0G. Low maturity. Likely early placeholder.
- **Lane overlap with Target Track 1:** Medium — concept overlaps but execution unclear and deploy chain is wrong.

### Claw_Wallet
- **Track:** 1 (Agent wallet infra + 0G Storage)
- **What it does:** Crypto wallet built for Agents. 2-of-3 Shamir Secret Sharing shard fragmentation. Encrypted share upload to 0G Storage. On-chain backup registration. Structured recovery receipts. The recovery architecture is self-custodial, trust-minimized, auditable, and built for autonomous AI agents.
- **Tech stack:** React, Solidity, 0g
- **Deploy ecosystem:** 0G
- **Progress during hackathon:** Smart contract deployed, local backup flow integrated, encrypted share upload to 0G Storage, on-chain backup registration, structured recovery receipts. In progress: full restore pipeline, frontend recovery interface, end-to-end demo polish.
- **Maturity signal:** Active development with completed milestones. Has demo video link. Multiple screenshots. This is a working-in-progress submission with real 0G integration.
- **Lane overlap with Target Track 1:** Medium — agent wallet/recovery is infra-adjacent but narrower than agent framework/skill/memory. Doesn't use 0G Compute. The "wallet for agents" lane is distinct from "agent orchestration framework."

### AgentHub
- **Track:** 1 or 3 (Agent marketplace / "AWS for AI agents")
- **What it does:** Secure execution platform + marketplace for third-party specialist AI agents. Platform-managed sandboxed execution with default-deny network access, scoped data handling, transparent permission disclosure, full audit logging, metered usage.
- **Tech stack:** React, Web3, Next, Vue, Rust, Node, Java — broad stack
- **Deploy ecosystem:** Base (not 0G mainnet)
- **Demo:** Has YouTube demo video. Project images showing UI.
- **Maturity signal:** Well-written description, clear business case, existing demo video. However deployed on Base not 0G — same disqualification risk as AgentMem.
- **Lane overlap with Target Track 1:** High concept overlap — "secure agent execution platform" — but deploying on Base severely weakens the 0G integration story.

### AgentMart
- **Track:** 1 or 3 (Agent services + memory)
- **What it does:** Decentralized platform where autonomous AI agents register, offer services, get hired for tasks, and maintain persistent long-term memory. Uses 0G Chain + 0G Storage for memory.
- **Tech stack:** Python, Solidity
- **Demo:** Has video (webm file)
- **Maturity signal:** Simple implementation (5-step workflow). Has 0G Chain + 0G Storage integration claimed. No project link visible.
- **Lane overlap with Target Track 1:** Medium — similar to the Ajently prior winner shape (agent marketplace). The "agent memory" component directly competes with the memory sub-lane.
